What Did We Learn From Building Our Wedding Palace Web Page

Summary of the Work

Building the Wedding Palace web site has given us an opportunity to learn a lot about organizing, designing, teaming and developing a web page. Our objective was to develop a clean and elegant web site for weddings that would allow brides to display bridal dresses, guests' dresses, and jewelry, with an ease of viewing through a simple and visually appealing format. We began by reading through all of the specifications which provided us with insight into the importance of creating six (6) web pages, completing mockups, validating, and reflecting. The planning phase was first and then came the mockups that will help us see exactly how each page should appear prior to writing a single line of code. This also helped prevent future errors while keeping our project very well organized.

Challenges We Faced and How These Were Overcome

Our home page was one of the most critical aspects of the project as it contained the registration form. The specification required that the form needed to validate the User's First Name, Last Name, Mobile Number and E-mail. Initially we experienced some problems with the correct functioning of the validations, specifically with the 8 digit mobile phone rule. To resolve these issues we simplified the JavaScript code and clearly defined the conditions of each field. After the validations were operational we were able to add a message for both successful and unsuccessful validation; this improved the usability of the form.

We also discovered how to utilize local storage to store the User's name on the Home Page and have it displayed on the Bride Dresses, Guest Dresses, and Jewelry Pages. When the User inputs his/her name on the Home Page, it displays automatically on the Bride Dresses, Guest Dresses, and Jewelry Pages. This was a new learning experience for us, and it enhanced the user-friendliness of the website. It also demonstrated how JavaScript can link together various web pages with no data base.

Another challenge we encountered was maintaining uniformity throughout the six (6) pages. In the beginning, each page was developed individually and therefore appeared somewhat differently. To resolve this issue, we developed strict guidelines for fonts, colors, navigation layouts and spacing. We chose to utilize the softer wedding colors such as ivory, blush and beige to fit the elegant theme and to accentuate the dresses. We designed the layout to be as simple as possible so that users could easily view the images and content without distractions. As soon as we incorporated the same designs onto all the pages, the website seemed more professional and cohesive.

What We Learned About Group Project Work

Working in groups taught us the significance of effective communication and task delegation. Each member of the group was responsible for developing individual pages and sections of the project. On occasion, our code conflicted or overlapped, however, we resolved the conflicts by reviewing each other's work and agreeing upon a common format. This experience taught us that planning and teamwork are equally important as technical skills.

What We Learned Technically

Learning to use Bootstrap was another significant aspect of this project. The Bootstrap Grid System enabled us to arrange images into equal columns without having to write extensive amounts of CSS. This made the dress and jewelry galleries look neat and balanced. We also utilized Bootstrap for Responsive Design, so the pages functioned effectively across different screen sizes. This was crucial as wedding websites need to be easy to navigate on cell phones.

From a technical standpoint, we learned how HTML, CSS, JavaScript, and Bootstrap are combined to develop a complete web site. We learned how to organize files, use external CSS to regulate the entire web site's design, and write basic JavaScript functions for validation and interactivity. We also learned the importance of testing every feature to ensure that the overall project operates smoothly. Developing mock-ups prior to writing any code taught us the importance of designing before developing, and ultimately made the completion of the final web site simpler.

Conclusion

In conclusion, this project helped us enhance our design abilities, coding knowledge, and teamwork abilities. We completed a six-page web site that features a clean layout, consistent designs, working form validation, and a customized user experience. This project exemplified the extent of planning, creativity, and problem solving involved in developing a commercial web site. We are pleased with the final product and the skills we acquired along the way.
